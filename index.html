Artifact 2: Algorithms and Data Structures

The artifact selected for this enhancement is a MongoDB-based dashboard application originally created during my CS 340 coursework. The project uses a MongoDB database containing animal shelter outcome data and a Python-based application layer to query and display filtered results. The artifact was originally developed to demonstrate basic CRUD functionality and database interaction. For this milestone, the artifact was enhanced to focus specifically on query execution behavior, algorithmic efficiency, and the impact of data structures such as indexes on performance.

I selected this artifact for my ePortfolio because it provides a clear and practical demonstration of algorithmic principles applied in a real-world database context. While the application itself is relatively simple, the underlying query behavior highlights important concepts related to algorithms and data structures, including linear search, indexing, and performance trade-offs.

The enhancement focused on analyzing how MongoDB executes queries when an index is not present. Using MongoDB Compass, I executed a filtered query on the animal_type field and examined the resulting execution plan. The explain plan showed a collection scan (COLLSCAN), meaning every document in the collection was examined to locate matching results. This behavior directly reflects a linear search algorithm, where time complexity increases proportionally with dataset size.

This artifact showcases my ability to analyze algorithmic efficiency, interpret execution metrics, and evaluate how data structures such as indexes can significantly improve performance. By identifying the absence of an index and observing its effect on query execution, the artifact was improved from a basic database interaction into a meaningful demonstration of algorithm and data structure concepts.

Enhancing this artifact reinforced the connection between theoretical algorithm concepts and practical implementation. Reviewing the explain plan made it clear that fundamental principles such as linear search and indexing still apply, even when a database abstracts the low-level mechanics. One challenge during this process was configuring the local MongoDB environment correctly, particularly managing authentication and service settings. Troubleshooting these issues strengthened my understanding of system configuration and tool usage.

Narrative Summary

This enhancement demonstrates how algorithmic behavior shows up in real database queries. By reviewing MongoDB execution output, I was able to connect a collection scan (COLLSCAN) to a linear search pattern, then show how indexes function as a data structure that improves performance. The key learning was seeing how time and resource usage scale when queries cannot use an index.

  • Skills demonstrated: execution plan analysis, performance reasoning, indexing as a data structure
  • Evidence: explain-plan screenshot and the milestone narrative

Artifact 3: Databases

The artifact selected for this enhancement is an Animal Shelter Dashboard originally created for CS 340. The original version demonstrated basic CRUD operations using MongoDB and displayed animal shelter data through a Python-based dashboard. For CS 499, this artifact was enhanced to better demonstrate database design principles, secure data access, and efficient data retrieval practices.

The enhanced artifact consists of two primary components:

  • MongoDB CRUD module (CRUD_Python_Module.py) that encapsulates database operations.
  • Dash-based interactive dashboard (Jupyter notebook) that retrieves, displays, and visualizes database records.

This artifact was selected for inclusion in my ePortfolio because it demonstrates my ability to design, implement, and enhance database-driven applications. It showcases practical database skills applicable to real-world development, including secure connections, structured data access, and performance-aware querying.

Narrative Summary

This enhancement focused on database practices that improve security, performance, and maintainability. I isolated database operations into a dedicated CRUD layer, validated connectivity early, and improved retrieval efficiency using projection and limiting. These changes demonstrate how database design decisions affect application behavior and scalability.

  • Skills demonstrated: database abstraction, secure access patterns, performance-aware querying
  • Evidence: indexes screenshot and the milestone narrative

Key Database Enhancements

  • Centralized CRUD abstraction: database access isolated behind a dedicated class for maintainability and scalability.
  • Secure MongoDB authentication: authenticated connection with connectivity validation at startup.
  • Efficient data retrieval: field projection and result limiting to reduce overhead and improve scalability.
  • Interactive visualization: filterable table and map visualization tied to database records.

What I Learned

Enhancing this artifact reinforced how database-level design decisions impact application performance and usability. Implementing projections and limits highlighted the importance of retrieving only the data an application needs. A major challenge was diagnosing and resolving authentication and connection issues, which strengthened my understanding of MongoDB security configuration and the value of validating connectivity early in the application lifecycle.
