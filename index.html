Artifact 2: Algorithms and Data Structures

This enhancement focused on improving query efficiency and analyzing algorithmic performance within the CS 340 animal shelter dashboard project. The original system relied on default query behavior without examining execution plans or implementing indexing strategies.

During the enhancement process, I analyzed MongoDB query execution using the explain() function. The results revealed a COLLSCAN (collection scan), which performs a linear search across all documents. This approach has O(n) time complexity and does not scale efficiently as data volume increases.

This enhancement demonstrates applied algorithmic principles, complexity analysis, and performance evaluation. It reflects my ability to evaluate computing solutions, identify inefficiencies, and implement improvements while understanding the trade-offs between query speed, storage overhead, and index maintenance costs.
