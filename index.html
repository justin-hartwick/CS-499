Artifact 2: Algorithms and Data Structures

This enhancement builds on the same CS 340 MongoDB dashboard project, but focuses on algorithmic efficiency and the impact of data structures on performance. Specifically, I analyzed how MongoDB executes filtered queries when an index is not available, and I used query execution plans to identify when the database was forced to scan the entire collection.

Using MongoDB Compass, I executed filtered searches and reviewed the explain plan output. When no index existed for the target field, the explain plan showed a COLLSCAN operation. This reflects linear search behavior, where performance scales proportionally with the number of documents in the collection. By identifying the missing index and documenting the results, the artifact demonstrates how indexing serves as an important data structure that improves efficiency and scalability.

This artifact aligns with course outcomes related to designing and evaluating computing solutions using algorithmic principles. It also demonstrates my ability to interpret execution metrics, explain trade-offs in design decisions, and communicate technical findings with clear evidence.
